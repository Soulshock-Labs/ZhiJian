# 模板识别与净空标准（v1.2.0）

适用项目：`smart-teacher-assistant`  
适用场景：国内幼儿园周计划 / 日教案 / 活动计划表等多样化 Word 模板  
目标：在**不破坏模板样式**的前提下，稳定识别“固定模板区”和“可填内容区”，并支持净空与回填。

**相对 v1.1.9 的变更摘要（v1.2.0）**

- **导出回填引擎**：周/日模板填充改为 **Aspose.Words**（与净空 lxml 引擎配合）；导出响应头含 `X-Export-Engine`，文档「备注」与自定义属性可核对引擎与版本（非 Aspose 评估水印）。

**相对 v1.1.8 的变更摘要（v1.1.9）**

- **周网格回填**：识别星期表头行与星期列，在净空骨架上向全部星期列分发内容；去重使用 lxml 单元格元素，不用 `id()`，避免 GC 后误判「已填」。
- **子行标签**：Word 多段拆字（如竖排 `集体` + `活动` → `集体\n\n活动`）与紧凑文本 `集体活动` 等价匹配，净空与 `match_field` 一致。

**相对 v1.1 的变更摘要（v1.1.8）**

- **保护区**：不再将「整表第一行」默认视为星期表头；改为 **星期表头行动态识别**（某行含 **≥3** 个 `星期X` / `周X` 表头单元格），仅该行全宽保护。若第一行是「活动主题」等元信息行而无星期列，**R0C1 等可填格可被净空**（与 `text大三班` 类模板一致）。
- **协议 5.3**：补充 `{{Tag}}` 在 Word 中被拆成多个 `<w:r>` 时的检测方式（拼合文本 + 字符区间）。
- **净空 6.4**：`weekly_grid_tagged` 下无 `{{Tag}}` 的逐天内容格按普通可填区清空。
- **报告**：增加 `day_header_rows` / 每表 `day_header_row` 字段便于验收。

**相对 v1.0 的变更摘要**

- 增加**语义词典同义扩展**与可选**匹配置信度权重**（用于报告与人工复核，非黑盒打分）。
- 增加 **`{{Tag}}` 强锚点协议**（占位符模板优先）。
- 增加**嵌套子行 / 二级标签**的映射说明（如户外活动下的「自主 / 集体」）。
- 增加**媒体与复杂对象**的保护区说明及实现边界。
- 增加**识别报告 JSON** 与**自动化验收指标**（结构偏离、误杀率、回填对齐）。

## 1. 设计原则

1. **结构优先，不依赖颜色**
   - 大部分园所模板没有真实底色（Word 深色模式仅视觉效果），不能把“有无颜色”作为主标准。
2. **首列与星期表头行是骨架锚点**
   - **第一列**通常是行标签区，默认保护。
   - **星期表头行**由动态规则识别（某行含 ≥3 个「星期一～五 / 周一～五」），该行整行保护；**不得**将「未含星期关键词的第一行」整体当作表头保护（避免误杀 R0C1 等活动主题值格）。
3. **标签驱动内容槽位**
   - 通过标签单元格定位可填内容槽位，避免按固定坐标硬编码。
4. **样式零破坏**
   - 净空与回填都只改文本，不改字体、段落、边框、行高、合并关系等格式。
5. **可解释与可回溯**
   - 每次识别应输出“命中模块 / 未命中模块 / 风险提示 / 置信度说明”。
6. **强锚点优先于弱推断**
   - 若存在 `{{...}}` 占位符或明确模块 ID，优先于纯关键词猜测。

## 2. 区域定义标准

### 2.1 保护区（Protected）

以下单元格默认不可清空、不可覆盖：

- **星期表头行（动态）**：在表格中扫描，若某一行有 **≥3** 个单元格文本匹配 `星期[一二三四五]` 或 `周[一二三四五]`，则将该行索引记为 `day_header_row`，**该行全部单元格**视为保护区（含「时间及活动环节」等左侧表头格）。
- **第一列全部单元格**（行标签列，如「生活活动」「户外活动」等）。
- **非**「星期表头行」的**第一行**：不按「整行保护」处理；其中仅首列、关键词命中格、色块格等按下列规则分别保护（避免 `活动主题` 行右侧值格被误保护）。
- 含结构表头关键词的单元格（分散在多格时）：
  - `星期`、`周一`~`周五`、`周次`、`时间`、`班级` 等（见词典）
- 含模块标签词的单元格（见第 4 节模块词典与同义词）
- **强信号**：真实底色单元格（`w:shd/@w:fill` 非 `AUTO/FFFFFF`，及引擎识别的主题色底纹）；园所粉/灰标签格优先靠此项保护。

**v1.1 扩展：媒体与复杂对象**

- **含图片的单元格**：默认视为保护区，净空时跳过（避免误删绘图锚点导致版式异常）。
- **浮动文本框 / 形状内主内容**：若内容不在表格单元格主 XML 流中，`python-docx` 级解析可能无法统一处理；标准上视为**需专项解析或人工确认**，净空引擎默认**不删除**未遍历到的对象。
- **实现边界（诚实约定）**：全量识别 Word 内所有 `Drawing` / `w:pict` / 浮动 `wps:wsp` 需要直接解析 `word/document.xml` 或扩展解析链；产品阶段可先做到「表格单元格内 run 级文本」的可靠保护，再迭代全文档对象扫描。

### 2.2 可填区（Fillable）

满足以下条件才可清空/写入：

- 不在保护区
- 被识别为模块内容槽位（含强锚点 `{{tag}}` 映射）
- 与标签单元格存在有效映射关系（右侧 / 下方 / 同主行的星期列交叉格 / 嵌套子行）

## 3. 文档类型识别标准

先分类，再套用对应规则：

### 3.1 周计划（`weekly_grid`）

命中特征（任一组合）：

- 出现 5 天列头：`星期一`~`星期五` 或 `周一`~`周五`
- 出现“第 X 周周计划”等标题语义
- 行标签常见：`生活活动`、`学习活动`、`游戏活动`、`区域活动`、`户外活动`、`家园互动`

### 3.2 日教案（`daily_plan`）

命中特征：

- `导入`、`过程`、`延伸`、`反思` 四维模块
- 常见伴随词：`观察要点`、`活动准备`、`活动目标`

### 3.3 通用活动计划（`general_plan`）

命中特征：

- `教学主题/活动主题`、`活动目标`、`活动准备`、`评价与反思`
- 结构常为纵向标签 + 右侧内容

## 5. 槽位映射标准

### 5.1 纵向模板映射

- **规则 A**：标签右侧单元格为主内容槽位
- **规则 B**：若右侧是标签，则取下方同列单元格
- **规则 C**：合并单元格按“主单元格”写入

### 5.2 周历横表映射

- 首列为主行标签（如学习活动）
- **星期表头行**由动态识别得到（见 2.1），不一定是第 0 行；该行含「周一～周五」列头
- 内容槽位 = `主行 × 星期列` 交叉格
- 未识别到星期列时，不强填，标记风险

### 5.3 `{{Tag}}` 强锚点协议（v1.1 新增）

**协议 A1 — 检测**

- 对单元格全文匹配正则：`\{\{\s*([^}]+?)\s*\}\}`（允许标签内外空格）。
- **Run 拆分**：Word 自动更正/拼写检查可能将 `{{tag}}` 拆成多个相邻 `<w:r>`（如 `{{`、`户外活动`、`}}`）。**检测时应先拼合同一段落内所有 `<w:t>` 文本再跑正则**；净空与回填时用**字符位置区间**（在拼合串上的起止下标）确定占位符边界，**不得**假设占位符在同一 run 内。
- 若命中，该单元格标记为 **强锚点槽位** `ContentSlot`，`slot_id` = 括号内规范化后的字符串（如 `outdoor`、`plan.outdoor` 由映射表约定）。

**协议 A2 — 优先级**

- 强锚点槽位**优先于**纯坐标或弱语义推断。
- 回填时：将 AI 输出按 `slot_id` 写入对应单元格；若同一文档多个同名 `{{slot_id}}`，按出现顺序或业务规则扩展（需在报告中声明策略）。

**协议 A3 — 净空**

- 净空时：**保留** `{{...}}` 占位符文本本身；仅删除占位符之外的教师填写内容（若有）。
- 若无额外内容，可保持单元格仅含占位符。

### 5.4 嵌套子行 / 二级标签（v1.1 新增）

典型场景：主行「户外活动」下再分「自主活动」「集体活动」等子行。

**识别策略（建议）**

1. 首列出现**纵向合并**（`vMerge`）时，进入「主模块上下文」。
2. 后续行若首列或次列为更短标签（如「自主」「集体」「自主活动」），视为**子模块**，槽位仍按「星期列 × 子行」交叉定位。
3. 映射状态允许为 **`partial_match`**：例如已识别 `plan.outdoor`，但检测到子标签 `[自主, 集体]`，需在报告中说明，供规则或人工确认是否拆分为 `plan.outdoor.autonomous` / `plan.outdoor.group`（产品可选，标准先允许部分匹配）。

## 6. 净空标准（Clean Template）

### 6.1 净空范围

- 仅清空 Fillable 槽位文本
- 不清空 Protected 区域
- 遵守 `{{Tag}}` 协议 A3

### 6.2 样式保留要求（强制）

- 不删除段落
- 不重建表格结构
- 不修改边框、合并、行高、列宽
- 保留字体、字号、颜色、段落间距、对齐方式

### 6.3 净空输出命名

- `原文件名_净空模板.docx`

### 6.4 `weekly_grid_tagged` 下无占位符的内容格（v1.1.8）

- 文档类型为 **`weekly_grid_tagged`**（全文存在至少一处 `{{...}}`）时，**并非每个可填格都有占位符**（例如「集中活动」「游戏活动」等逐天列格可能仍为教师手写内容）。
- 此类单元格若**不含** `{{Tag}}` 子串（拼合 run 后检测），一律视为 **普通可填区**，执行与普通 `weekly_grid` 相同的**全文清空**（样式保留策略见 6.2），不因文档中存在其他占位符而跳过。

## 7. 回填标准（Generate）

1. 先完成模块识别，再执行写入
2. 仅写 Fillable 槽位，不动 Protected 区域
3. 写入策略：
   - 命中模块 → 写入
   - 未命中模块 → 保留原样 + 风险报告
4. 不允许“覆盖式重排”（禁止重建表格或大段替换）
5. **智能拆分（可选产品能力）**：若 AI 输出为长文而模板为多格分领域（如健康/语言），需单独规则或二次结构化输出；**不作为 v1.1 标准强制项**，避免误拆导致语义错位

## 8. 识别报告与置信度（v1.1 新增）

每次分析或生成宜返回机器可读报告，便于前端展示与回归。

**示例 JSON（字段可扩展）**

```json
{
  "status": "success",
  "confidence_score": 0.85,
  "doc_type": "weekly_grid",
  "day_header_rows": [1, -1],
  "mappings": [
    {
      "tag": "plan.outdoor",
      "status": "partial_match",
      "reason": "found nested sub-rows: [自主, 集体]"
    }
  ]
}
```

- **`day_header_rows`**：与 `document.xml` 中表格顺序一致；每个元素为该表的星期表头行 **0-based** 行号，**`-1`** 表示未识别到（该表无「≥3 个星期列」的行）。用于验收与调试，避免「假首行保护」。

**`status` 建议枚举**

- `matched`：槽位与模块明确对应
- `partial_match`：模块识别到但结构未完全展开（如嵌套子行）
- `unmatched`：无对应标签或无法映射
- `skipped_protected`：命中保护区，跳过

**`confidence_score`**

- 综合：词典命中权重、是否仅有弱关键词、是否存在星期列、是否存在嵌套未解析等因素的**启发式分数**（0~1），供产品展示，**不承诺**与真实准确率一一对应。

## 9. 自动化验收指标（v1.1 新增）

将“感性评估”转为可重复执行的检查项：

| 指标 | 含义 | 建议判定 |
|------|------|----------|
| **结构偏离度（Structure Shift）** | 净空前后对比 `document.xml` 中表格数量、`tbl` 子树规模或关键属性 | 表格数量变化 → Failed；合并关系异常 → Failed |
| **内容误杀率（False Clear）** | 保护区（星期表头行、首列、标签格、色块格）是否被清空 | 任一误清空 → Failed |
| **回填对齐率（Fill Alignment）** | 抽检槽位：填入文本是否极端撑破版式（可选：字符数阈值 + 人工抽检） | 作为回归集抽检项，不设绝对自动化阈值亦可 |

---

## 10. 质量验收标准（建议）

- 模块识别召回率（核心模块）>= 95%（**目标**，需样本集验证）
- 固定区误改率 <= 1%
- 表格结构一致性：行数 / 列数 / 合并关系在净空后与原版一致
- 失败可解释：返回未命中模块与原因

---

## 11. 风险与边界

以下场景需人工确认或二次规则：

- 标签完全自定义且无语义（如“栏目一/栏目二”）
- 大量图片/文本框承载主要内容（非表格）
- 复杂跨页套表 + 多层嵌套合并
- 同一模板同时混合「周计划 + 日教案」多种结构

---

## 12. 建议实施路线

1. 固化本标准到后端接口（如 `/template-standard` 版本号与本文同步）
2. 新增模板分析接口（建议）：`/template/analyze`
   - 返回：文档类型、命中模块、槽位计数、风险提示、`mappings` 与 `confidence_score`
3. 前端增加「识别报告卡」后再生成 / 净空
4. 建立真实模板样本集（持续扩展词典与规则）
5. **回归集**：至少包含：传统横表、嵌套子行周计划、含 `{{}}` 占位符模板

---

## 13. 版本结论

对于国内幼儿园模板高度异构的现实，推荐标准是：

**以首列、星期表头行（动态）与语义标签为锚点，辅以 `{{Tag}}` 强锚点与嵌套子行规则的结构识别标准**，  
颜色与纯坐标仅作辅助，不作为唯一依据。
